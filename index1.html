Essentials of Data Communication & Networking

Concise guide covering OSI/TCP-IP, devices, media and security — ready for notes or deployment.

What is Data Communication?

Data communication = exchange of data between devices over a medium (copper, fiber, wireless). Elements: sender, receiver, medium, message, protocol.

OSI Model — Layered Approach

OSI splits networking into 7 layers to standardize responsibilities and improve interoperability.

Layer 7 — Application

HTTP, FTP, SMTP — app-level services.

Layer 6 — Presentation

Formatting, encryption (TLS), compression.

Layer 5 — Session

Session/dialog control.

Layer 4 — Transport

TCP/UDP — reliability, flow control.

Layer 3 — Network

IP — routing and addressing.

Layer 2 — Data Link

Framing, MAC addressing (Ethernet).

Layer 1 — Physical

Electrical/optical/radio signals.

Tip

TCP/IP model = 4 layers (App, Transport, Internet, Link).

TCP/IP & Key Protocols

  • IP — addressing/routing (IPv4/IPv6)
  • TCP — reliable transport
  • UDP — low-latency datagrams
  • ICMP — diagnostics (ping)
  • HTTP/HTTPS/DNS — common app protocols

Topologies & Devices

Topologies

Bus, star, ring, mesh — choose by cost & fault tolerance.

Devices

Hub (L1), Switch (L2), Router (L3), Firewall.

Switching

Circuit vs Packet switching — internet uses packets.

Wireless

APs, channels, interference, 802.11 standards.

Transmission Media

  • Twisted pair — Ethernet Cat5e/Cat6
  • Coaxial — legacy cable
  • Fiber — high bandwidth, long distance
  • Wireless — shared medium, variable latency

Network Security Basics

  • Encryption (TLS, IPsec) for confidentiality
  • Integrity via hashes/signatures
  • Authentication with certs/keys
  • Segmentation: VLANs/subnets
